Diagrama de Caso de Uso

Um Diagrama de Caso de Uso é uma ferramenta visual que descreve a funcionalidade de um sistema do ponto de vista do usuário (ou "ator"). Ele mostra:

  • Atores: Quem ou o quê interage com o sistema.
  • Casos de Uso: As tarefas específicas que os atores podem realizar com o sistema.
  • Relacionamentos: Como os casos de uso se relacionam entre si e com os atores (ex: inclusão, extensão, generalização).

Em resumo, ele responde à pergunta: "O que o sistema faz?" e para quem. É útil para entender os requisitos do sistema e definir seu escopo.

Diagrama de Classes

Um Diagrama de Classes é um tipo de diagrama estrutural da UML (Linguagem Unificada de Modelagem) que descreve a estrutura estática de um sistema, mostrando suas classes, atributos, métodos e relacionamentos entre elas. Ele mostra:

  • Classes: Representam os objetos do sistema e seus dados.
  • Atributos: As características dos objetos (ex: nome, idade).
  • Métodos: As ações que os objetos podem realizar (ex: calcular, exibir).
  • Relacionamentos: Como as classes se relacionam entre si (ex: associação, herança, agregação, composição).

Em resumo, ele responde à pergunta: "Como o sistema é estruturado?". É útil para modelar a arquitetura do software e facilitar a comunicação entre desenvolvedores.
